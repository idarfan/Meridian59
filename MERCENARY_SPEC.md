# 自製佣兵 NPC — KOD Mod 規格

## 目標

新增一個佣兵 NPC class（暫名 `Mercenary`），行為需求依優先序：

1. **跟隨**：綁定一名主人（玩家），主人移動時跟隨，換房間（room）時跟著傳送過去
2. **補血**：主人 HP 低於門檻（預設 70%）時，對主人治療
3. **自動打怪**：主人附近出現敵對怪物、或主人受到攻擊時，主動攻擊該怪物
4. **佣兵不死**：佣兵可被怪物攻擊（可幫主人吸引仇恨），但**不可被殺死**——
   HP 歸零前觸發保護（HP 永不低於 1，或高速自動回血），永遠不進入死亡流程

## 非目標（第一版不做）

- 雇用/薪資/商店購買機制（admin console 直接生成即可）
- 佣兵裝備系統
- 多佣兵編隊 AI
- 佣兵升級/成長

## 實作原則

- **先調查、後實作**：動手寫 code 前，必須先完成下方「原始碼調查」並回報，
  經確認後才開始實作
- 最大化重用既有 class：用繼承而非複製貼上
- 所有數值（跟隨距離、補血門檻、治療量、攻擊力）定義為 class 常數或
  properties，方便調整
- 一次一個功能：跟隨 → 補血 → 打怪，分三個階段實作，每階段獨立測試通過
  才進下一階段

## 原始碼調查（Phase 0，唯讀）

從 kod/ 找出並回報：

1. **Monster 基底**：kod/object/active/holder/nomoveon/battler/monster/
   底下的繼承結構；哪個既有 class 最適合當佣兵基底
   （候選：會戰鬥的 faction soldier / guard 類、會跟隨的 pet / familiar /
   animate 類——確認實際存在哪些）
2. **跟隨機制**：既有「NPC 跟隨玩家」的實作在哪（animate 出來的怪物如何
   跟著施法者？）；跨房間跟隨怎麼做（監聽主人離開房間的訊息為何）
3. **移動/AI 迴圈**：monster 的移動與 AI 是靠什麼驅動
   （timer？收到什麼 message？），tick 間隔多少
4. **戰鬥介入**：怪物如何選定攻擊目標（仇恨/target 屬性名）；
   「主人被攻擊」時佣兵如何得知（戰鬥相關的 broadcast message 有哪些）
5. **治療手段**：對玩家加 HP 的正確做法——是 send 什麼 message 給
   user 物件（查 user.kod 的 HP 相關 message），或重用某個治療法術的邏輯
6. **攻擊型態與傷害類型**：monster 的攻擊如何定義傷害類型
   （物理/魔法/屬性——查 KOD 的 attack type / spell power 相關屬性）；
   既有怪物中誰會遠程魔法攻擊（如施法類怪物），其實作方式為何；
   怪物的抗性/免疫機制怎麼判定（哪個屬性決定「物理打不動」）；
   **定身/暈眩類控場法術**在 KOD 中如何實作（查 Hold 相關 spell——
   遊戲中有讓目標無法移動的法術，找出其效果 message 與持續時間機制），
   以及 NPC/monster 能否對怪物施放該效果
7. **經驗值歸屬**：怪物死亡時經驗/戰利品如何分配（查 monster 死亡處理的
   KOD——按傷害比例？最後一擊？擊殺者獨得？）；確認「佣兵造成的傷害」
   預設會歸給誰，以及把 credit 轉給主人的可行做法
8. **生成方式**：admin console 生成一隻怪並指定房間座標的指令語法；
   如何在生成時把主人（玩家物件）綁進佣兵的 property
9. **編譯與部署**：新增一個 .kod 檔需要動哪些 makefile / include；
   編譯後 .bof/.rsc 部署到 server 與 client 的流程
   （注意：**client 端 .rsc 更新後，記得跑 scripts/sync-rsc.sh 同步**，
   避免重演 Part 2-3 的資源缺失問題）

## 行為規格

### 跟隨（Phase 1）
- property：`poMaster`（主人玩家物件）
- 與主人距離 > N 格時朝主人移動；距離 ≤ 2 格時停止（不要貼臉）
- 主人離開房間：佣兵直接傳送到主人所在新房間（不用尋路跨圖）
- **主人死亡**：主人重生後，佣兵立即傳送到主人重生位置身邊
  （實作提示：這與跨房間跟隨是同一機制——監聽主人的房間變化即可涵蓋
   死亡重生的傳送；Phase 0 第 2 項調查時一併確認死亡重生是否會觸發
   同樣的房間變化訊息，若不會則需額外監聽死亡/重生事件）
- 主人登出：佣兵原地待命（第一版從簡）

### 補血（Phase 2）
- 每個 AI tick 檢查主人 HP；低於 `piHealThreshold`（預設 70%）時治療
- 治療量 `piHealAmount`（預設 50 HP），冷卻 `piHealCooldown`（預設 2 秒）
- **治療原則：寧可超補、不可少補**——補到超過 HP 上限就截斷至滿血即可
  （實作時確認 user 的 HP 加值 message 會自動 clamp 到上限，不會溢出出錯）
- 治療時對房間 broadcast 一則訊息（如 "佣兵對你施放治療"），
  資源字串記得進 .rsc

### 打怪（Phase 3）
- 觸發條件（任一）：主人被怪物攻擊 / 主人攻擊某怪物 / 敵對怪物進入
  佣兵視野 N 格內
- 目標優先序：攻擊主人的怪 > 主人正在打的怪 > 最近的敵對怪
- 主人 HP 低於門檻時，補血優先於攻擊
- **經驗歸屬**：佣兵對怪物造成的傷害/擊殺，經驗值必須歸主人所有
  （實作方式依 Phase 0 第 7 項調查結果決定：把佣兵傷害記在主人名下、
  或在怪物死亡結算時把佣兵的 credit 轉給 poMaster）
- **攻擊型態**：佣兵需具備物理與魔法兩種攻擊；偵測到目標物理抗性高
  （或物理攻擊持續未造成傷害）時切換魔法攻擊
  （實作方式依 Phase 0 第 6 項調查結果決定：最簡做法是攻擊自帶
  雙傷害類型或直接用魔法系傷害，能打動就好，不強求華麗施法動畫）
- **定身控場**：佣兵會施放定身法術（Hold 類效果，使目標無法移動）。
  施放時機優先序：正在攻擊主人的怪 > 同時被多隻怪圍攻時定住其中一隻。
  冷卻 `piHoldCooldown`（預設 10 秒）、持續時間依 Phase 0 調查的
  既有 Hold 效果機制（若可調，預設 8 秒）
- 不攻擊玩家（PvP 不參與）、不攻擊其他佣兵、**不對玩家施放定身**

### 數值（第一版，可調）
- **外觀**：女性人形 NPC——重用既有女性 NPC 的圖像資源
  （Phase 0 第 8 項調查時一併確認：既有女性 NPC/女性玩家模型的
  .bgf 圖像資源名稱與 KOD 中指定外觀的屬性，直接沿用即可，不做新美術）
- HP：5000（遠高於任何區域怪物的輸出）、攻擊力：比照中階怪物（調查後定）、
  移動速度：與玩家相近
- **不死保護**（雙重保險）：受傷處理中攔截——若本次傷害會使 HP ≤ 0，
  將 HP 定為 1 並跳過死亡流程；另每個 AI tick 自動回血 100
- Phase 0 調查時確認 monster 死亡流程的進入點（收到什麼 message 判定死亡），
  保護邏輯要攔在該點之前

## 測試計畫

每 Phase 完成後：
1. 編譯 KOD、部署 .bof/.rsc（含 client 端同步）、重啟 server
2. admin 生成佣兵綁定測試角色，逐項驗證該 Phase 行為
3. 回報測試結果表格，確認後才進下一 Phase

## 版控

- 每個 Phase 一個 commit，訊息格式：`[merc] Phase N: <摘要>`
- 這是本地 mod，不推 upstream
